# Security Policy

## Reporting a vulnerability

Please do not disclose security issues publicly before review.

Report vulnerabilities privately to the project maintainers with:

- affected contracts or files
- reproduction steps
- impact assessment
- suggested remediation if available

## Scope

The intended scope includes:

- escrow contracts
- factory contracts
- token handling logic
- timeout and dispute paths
- SDK calls that could create unsafe integration assumptions

## Important note

No smart contract system should be used in production without dedicated testing and independent review.
