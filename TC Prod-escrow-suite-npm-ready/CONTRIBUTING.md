# Contributing

Thanks for contributing.

## Workflow

1. Fork the repository
2. Create a feature branch
3. Add or update tests
4. Run build, lint, and test locally
5. Open a pull request

## Standards

- Keep Solidity contracts small and explicit
- Prefer deterministic state transitions
- Add tests for every new state-changing path
- Avoid hidden side effects
- Document security implications for new features

## Pull requests

Every pull request should include:

- a short summary of the change
- the motivation for the change
- test coverage notes
- any security considerations
